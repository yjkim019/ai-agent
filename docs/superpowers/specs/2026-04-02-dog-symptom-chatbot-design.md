# 강아지 증상 분석 챗봇 — 설계 문서

**작성일**: 2026-04-02
**대상 과정**: 에이전트 개발 및 평가 마스터 코스 (1~4주차)

---

## 프로젝트 개요

견주들이 반려견의 이상 증상을 발견했을 때 방치하지 않도록, 단계적 질문을 통해 증상을 좁혀가고 긴급도·의심 질환·대처법·병원 방문 권고를 담은 종합 리포트를 제공하는 챗봇.

### 핵심 요구사항

| 항목 | 내용 |
|---|---|
| 대상 동물 | 강아지 (개) |
| 데이터 소스 | 수의학 PDF 문서 기반 RAG |
| 대화 방식 | 단계적 질문 흐름 (에이전트가 추가 질문으로 증상 좁혀가기) |
| 출력 형식 | 종합 리포트 (긴급도 + 의심 질환 + 가정 대처법 + 병원 권고) |
| 외부 도구 | 동물병원 위치 검색 + 품종별 특성 조회 |

---

## 아키텍처 전략: StateGraph 파이프라인

`create_agent`를 오케스트레이터로 유지하면서, 복잡한 로직은 `StateGraph`로 구현한 뒤 `@tool`로 래핑하여 붙이는 방식. 주차별로 기능을 점진적으로 쌓아가는 구조.

```
create_agent (오케스트레이터)
    ├── @tool: symptom_pipeline   ← StateGraph (단계적 질문 + RAG 검색)
    ├── @tool: get_breed_info     ← 품종별 취약 질환 조회
    └── @tool: find_nearby_vet   ← 동물병원 위치 검색
```

## 1주차: 기초 에이전트 웹서비스

### 목표
채팅 UI에서 강아지 증상을 입력하면 기초 분석 답변이 나오는 웹서비스 완성.

### 구성 요소

```
사용자 입력
    ↓
create_agent
    ├── Tool: search_symptoms   ← ES BM25 텍스트 검색 (dog-symptoms 인덱스)
    ├── Tool: get_breed_info    ← 품종별 취약 질환 조회 (신규)
    └── Tool: find_nearby_vet  ← 동물병원 검색 (기존 코드 재활용)
    ↓
자유형 답변 반환
```

### 작업 목록

| 작업 | 내용 |
|---|---|
| ES 인덱스 생성 | `dog-symptoms` 인덱스 — 증상/질환 문서 BM25 검색용 |
| `get_breed_info` 도구 구현 | 품종명 입력 → 취약 질환 목록 반환 |
| `find_nearby_vet` 재활용 | 기존 코드 도메인만 변경 |
| 시스템 프롬프트 작성 | 강아지 증상 상담사 역할 정의, 과도한 진단 금지 면책 조항 포함 |
| 샘플 데이터 적재 | 10~20개 증상-질환 매핑 JSON → ES 수동 적재 |

### 산출물
채팅 UI에서 "강아지가 밥을 안 먹어요" 입력 시 기초 분석 답변 반환.

## 2주차: Opik 관찰성 + 평가

### 목표
에이전트가 증상을 올바르게 분석하는지 수치로 측정하고 취약점을 도출.

### Opik 트레이싱 구조

```
사용자 입력
    ↓ [Trace 시작]
    ├── LLM 판단 로그      ← 어떤 도구를 선택했는가
    ├── Tool 호출 로그     ← 각 도구 입력/출력
    └── 최종 답변 로그     ← 생성된 답변 내용
    ↓ [Trace 종료] → Opik 대시보드 전송
```

### 평가 지표

| 지표 | 측정 내용 | 방법 |
|---|---|---|
| 답변 정확성 | 증상에 맞는 질환을 언급했는가 | LLM-as-a-judge (1~5점) |
| 긴급도 정확성 | 응급 증상을 응급으로 분류했는가 | 규칙 기반 Heuristic |
| 도구 호출 적절성 | 불필요한 도구를 호출하지 않았는가 | Tool call 로그 분석 |
| 할루시네이션 | 존재하지 않는 약/치료법을 언급했는가 | LLM-as-a-judge |

### 평가 데이터셋 형식

```json
[
  {
    "input": "강아지가 갑자기 쓰러졌어요",
    "expected_urgency": "즉시 병원",
    "expected_tools": ["search_symptoms"]
  },
  {
    "input": "말티즈가 가끔 재채기를 해요",
    "expected_urgency": "가정 관찰",
    "expected_tools": ["search_symptoms", "get_breed_info"]
  }
]
```

### 산출물
Opik 대시보드에서 에이전트 취약점(어떤 증상 유형에서 오답이 나오는지)을 시각화한 분석 리포트.

## 3주차: StateGraph + PDF RAG 고도화

### 목표
단계적 질문으로 증상을 좁혀가고 PDF 지식베이스 기반 종합 리포트를 생성하는 완성형 RAG 서비스.

### StateGraph 흐름

```
START
  ↓
[증상 수집]   ← 사용자 첫 입력
  ↓
[심화 질문]   ← 부족한 정보 수집 (최대 3회, 충분하면 스킵)
  ↓             질문 예시: "언제부터인가요?", "밥은 먹나요?", "다른 증상도 있나요?"
[RAG 검색]   ← PDF 지식베이스 하이브리드 검색 (BM25 + kNN + RRF)
  ↓
[리포트 생성] ← 긴급도 + 의심 질환 + 대처법 + 병원 권고 구조화
  ↓
END
```

### PDF 파이프라인 (기존 scripts/ 재활용)

```
수의학 PDF 수집          → 01_download_pdfs.py
텍스트 추출 + 청킹        → 02_parse_and_chunk.py  (500토큰, 50 오버랩)
OpenAI 임베딩 생성        → 03_generate_embeddings.py
ES 인덱스 적재            → 04_index_to_es.py  (dog-knowledge 인덱스)
하이브리드 검색 (BM25+kNN+RRF)
```

수집 대상 PDF:
- 반려견 가정 간호 가이드
- 품종별 유전 질환 자료
- 응급 증상 판단 기준표

### 종합 리포트 출력 형식

```
🔴 긴급도: 24시간 내 병원 방문 권장

📋 의심 질환
  1. 췌장염 (일치도 높음) — 구토 + 식욕부진 + 복통 증상과 일치
  2. 위장염 (일치도 중간) — 구토 단독 증상 시 가능

🏠 가정 내 대처법
  - 12시간 금식 후 소량씩 물 제공
  - 구토 횟수/색깔 기록해두기

🏥 병원 방문 시 전달사항
  - 증상 시작: 2일 전
  - 구토 횟수: 하루 4회 이상
```

### 산출물
PDF 지식베이스 기반 단계적 질문 → 종합 진단 리포트 생성 완성형 서비스.

## 4주차: Deep Agent (서브에이전트 병렬 분석)

### 목표
복합 증상을 서브에이전트가 병렬로 분석하고 반려견 이력을 기억하는 지능형 진단 서비스.

### 전체 아키텍처

```
사용자: "진돗개 7살, 3일째 구토+식욕부진+무기력증"
              ↓
     create_agent (오케스트레이터)
              ↓
    복잡한 증상 감지 → task() 병렬 위임
        ↙            ↓            ↘
[증상분석          [품종리서치       [응급판단
 서브에이전트]      서브에이전트]     서브에이전트]
BM25+Vector       진돗개 취약       긴급도 점수
RAG 검색          유전 질환         계산
        ↘            ↓            ↙
                  결과 병합
                     ↓
               VFS에 중간 결과 저장
                     ↓
               최종 종합 리포트 생성
                     ↓
               동물병원 위치 검색
```

### 단순/복잡 증상 분기

| 조건 | 처리 방식 |
|---|---|
| 증상 수 ≤ 2개 | 3주차 StateGraph 파이프라인 (빠름) |
| 증상 수 ≥ 3개 또는 복합 상황 | task() 서브에이전트 병렬 위임 (정밀) |

### VFS 파일 구조

```
/vfs
  ├── symptom_analysis.json   ← 증상분석 서브에이전트 결과
  ├── breed_risk.json         ← 품종리서치 서브에이전트 결과
  ├── urgency_score.json      ← 응급판단 서브에이전트 결과
  └── final_report.md         ← 오케스트레이터 최종 종합 리포트
```

### 장기 메모리 구조

SQLite 대화 지속성(3주차) + 반려견 프로필 기억:
- 이름, 품종, 나이, 체중
- 과거 진단 이력
- 알레르기 / 복용 중인 약

활용 예시:
> "지난번에 췌장염 진단받은 적 있으시죠? 이번 증상과 연관성이 있을 수 있습니다."

### 산출물
복잡한 복합 증상을 서브에이전트가 병렬 분석하고 반려견 이력까지 기억하는 지능형 진단 서비스.

- **런타임**: Python 3.11~3.13, uv
- **웹 프레임워크**: FastAPI + Uvicorn
- **AI/LLM**: LangChain, LangGraph, OpenAI GPT-4o
- **임베딩**: OpenAI text-embedding-3-small
- **검색**: Elasticsearch (BM25 + kNN + RRF)
- **ReRank**: Cohere rerank-v3.5
- **체크포인트**: LangGraph-Checkpoint-SQLite
- **관측성**: Opik
